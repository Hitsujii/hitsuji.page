Hello!!

I started this blog because I wanted a place to write about learning C++.

That was the plan, anyway.

I thought I would set up a small site, write a quick first post, and then go back to the actual thing I wanted to do: opening my editor and writing some C++ code.

Unfortunately, computers had other plans.

## The problem

Before I wrote any meaningful C++, I somehow ended up opening a React course.

I do not fully know how that happened.

One moment I was thinking about variables, compilers, headers, and all the usual beginner C++ things.

The next moment I was adjusting the blog layout, thinking about components, changing small details, and convincing myself that this was somehow part of the learning process.

To be fair, it kind of was.

But also, not really.

## Getting ready

Getting ready to learn can look a lot like learning.

You make the folder. You pick the theme. You organize the notes. You improve the website. You make everything feel nice and clean and ready.

And then you notice that you have spent the whole evening preparing to start.

This is a very dangerous activity because it feels productive.

It is not doomscrolling. It is not obviously useless. It has buttons, files, commits, and maybe even a README.

That makes it harder to admit that you are still avoiding the uncomfortable part.

The part where you are new at something.

## The detour

Instead of choosing a simple template and moving on, I went straight into Next.js.

Then I started tweaking things.

Then I tweaked the things around the things.

Then the small blog setup became a small blog starter.

That detour eventually turned into [NextPaper](https://github.com/Hitsujii/next-paper), a Next.js blog starter inspired by [AstroPaper](https://github.com/satnaing/astro-paper) and built on top of [Tailwind Nextjs Starter Blog](https://github.com/timlrx/tailwind-nextjs-starter-blog).

Was this necessary for learning C++?

No.

Was it fun?

Yes.

Do I regret it?

Not really.

## So what now?

Now the blog exists.

That means I can finally use it for the thing it was supposed to be for.

I want this place to be a small record of learning C++ properly. Not pretending to know things. Not skipping the boring parts. Not only collecting syntax and hoping that understanding appears later.

Just learning, writing, making mistakes, fixing them, and writing down what happened.

This first post is not really a tutorial.

It is more of a warning to myself.

The setup is done now.

Probably.

Hopefully.

Thank you for reading!

**This is my first post, and the first mistake I am documenting.**
